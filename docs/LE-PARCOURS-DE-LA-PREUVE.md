# LE PARCOURS DE LA PREUVE

## Objet

Ce document décrit le principe fondamental qui guide l'ensemble de l'architecture d'InsouWiki.

L'objectif d'InsouWiki n'est pas de fournir des réponses.

Son objectif est d'accompagner le lecteur jusqu'aux preuves qui lui permettront de construire sa propre compréhension.

Le parcours de la preuve constitue ainsi la colonne vertébrale du projet.

---

# Une question

Toute exploration documentaire commence par une question.

Cette question peut être précise :

* « Que dit cet auteur sur ce sujet ? »

Ou plus ouverte :

* « Que puis-je découvrir aujourd'hui ? »

Dans tous les cas, elle exprime une intention du lecteur.

---

# Une intention

L'intention oriente naturellement le début de l'exploration.

Elle peut consister à :

* comprendre ;
* comparer ;
* vérifier ;
* retrouver un contexte ;
* suivre une évolution ;
* découvrir.

Ces intentions constituent les principales expériences documentaires proposées par InsouWiki.

---

# Une porte d'entrée

L'intention conduit vers une porte d'entrée documentaire.

Cette porte d'entrée peut être :

* un sujet ;
* un auteur ;
* une citation ;
* une affirmation ;
* une évolution ;
* une convergence ;
* un document.

Elle représente le premier contact entre le lecteur et le patrimoine documentaire.

---

# Une exploration

À partir de cette porte d'entrée, InsouWiki construit une exploration documentaire.

Cette exploration organise progressivement les éléments les plus pertinents.

Elle ne constitue jamais une réponse.

Elle prépare un parcours.

---

# Des observations

L'exploration fait apparaître des observations documentaires.

Par exemple :

* une continuité ;
* une évolution ;
* une convergence ;
* une divergence.

Ces observations permettent d'orienter la lecture.

Elles demeurent toujours reliées aux éléments qui les justifient.

---

# Des connaissances

Les observations conduisent vers des connaissances documentaires.

Ces connaissances organisent les faits sans jamais s'en détacher.

Elles facilitent la compréhension progressive du sujet.

---

# Des faits

Les connaissances renvoient ensuite aux faits documentaires.

Les faits constituent les premiers éléments directement observables dans les documents.

Ils représentent les fondations de tout raisonnement documentaire.

---

# Des séquences

Chaque fait renvoie vers une ou plusieurs séquences documentaires.

Les séquences permettent de retrouver le passage précis dans lequel le fait est observable.

---

# Les documents

Les séquences conduisent naturellement aux documents d'origine.

Le lecteur retrouve alors les vidéos, les textes ou les enregistrements qui constituent le patrimoine documentaire.

---

# Les preuves

Le parcours s'achève par les preuves.

Ou plus exactement...

Il retrouve les preuves qui étaient présentes depuis le début.

InsouWiki ne crée jamais les preuves.

Il aide simplement le lecteur à les atteindre.

---

# Le rôle de l'intelligence artificielle

L'intelligence artificielle intervient tout au long du parcours.

Elle facilite :

* l'organisation ;
* la découverte ;
* les rapprochements documentaires ;
* les observations.

Elle ne remplace jamais les preuves.

Elle réduit simplement la distance qui sépare le lecteur de celles-ci.

---

# Une architecture au service du lecteur

Toute l'architecture d'InsouWiki découle de ce parcours.

Chaque objet du domaine, chaque raisonneur documentaire, chaque scénario de référence et chaque choix technique ont pour vocation de rendre ce chemin :

* plus simple ;
* plus transparent ;
* plus progressif ;
* plus vérifiable.

---

# Vision

Le patrimoine documentaire existe indépendamment d'InsouWiki.

Les preuves existent déjà.

Les documents existent déjà.

La mission d'InsouWiki n'est pas de créer un savoir nouveau.

Elle consiste à rendre le chemin vers les preuves plus accessible.

Ainsi, le lecteur ne reçoit pas une conclusion.

Il accomplit lui-même un parcours documentaire qui lui permet de comprendre, de vérifier et de juger librement.

C'est ce parcours qui constitue la véritable valeur d'InsouWiki.
