# ÉTAT DU PROJET

*Version de référence – Fin du Sprint 9.5*

---

# Vision

La vision d'InsouWiki est désormais stabilisée.

Le projet vise à construire un patrimoine documentaire vérifiable, traçable et durable à partir de sources primaires.

L'intelligence artificielle y est utilisée comme un outil d'organisation documentaire, sans jamais se substituer au jugement du lecteur.

---

# Domaine documentaire

Le langage documentaire d'InsouWiki est désormais largement établi.

Les principaux concepts du domaine sont :

* Sources documentaires
* Documents
* Transcriptions
* Séquences documentaires
* Faits documentaires
* Relations documentaires
* Connaissances documentaires
* Sujets documentaires
* Évolutions documentaires

Ces concepts constituent le cœur du projet.

Ils évolueront encore, mais leurs fondements sont considérés comme stables.

---

# Architecture

L'architecture suit désormais un principe clair :

Vision

↓

Expériences lecteur

↓

Raisonnements documentaires

↓

Domaine

↓

Tests

↓

Code

Le domaine constitue le centre de l'application.

Le code en est l'implémentation.

---

# Documentation

Le projet possède désormais une documentation structurée :

* documents de vision ;
* documents de domaine ;
* méthodes documentaires ;
* ADR ;
* journaux de développement.

Chaque catégorie possède un rôle clairement identifié.

---

# Développement

Les principaux composants documentaires sont en place.

La collecte documentaire, les transcriptions, les faits, les relations, les connaissances et les premiers raisonnements documentaires disposent de leurs premiers modèles.

Les raisonnements documentaires restent volontairement prudents.

Ils privilégient l'absence de conclusion à une conclusion insuffisamment fondée.

---

# Qualité

Le projet est accompagné d'une suite de tests automatisés couvrant les principaux concepts du domaine.

Les tests constituent également une documentation des règles documentaires.

Ils protègent durablement les principes fondamentaux du projet.

---

# Décisions stabilisées

Les principes suivants sont désormais considérés comme fondateurs :

* les sources documentaires sont immuables ;
* le patrimoine documentaire repose sur des sources primaires ;
* la preuve documentaire précède toujours le raisonnement ;
* les expériences lecteur guident l'évolution du domaine ;
* le domaine guide les tests ;
* les tests guident le code.

---

# Ce que le projet refuse

InsouWiki refuse notamment :

* de produire des conclusions sans preuves ;
* de rompre la traçabilité documentaire ;
* de confondre faits, connaissances et raisonnements ;
* de privilégier la rapidité au détriment de la vérifiabilité ;
* de remplacer le jugement du lecteur.

# Priorité actuelle

Entrer dans la seconde phase du projet :

Construire progressivement les raisonnements documentaires permettant au lecteur d'explorer, comparer et comprendre un patrimoine documentaire fondé exclusivement sur des preuves vérifiables.

---

# Conclusion

À l'issue du Sprint 9.5, InsouWiki possède des fondations documentaires, architecturales et méthodologiques solides.

Les prochaines évolutions ne consisteront plus principalement à créer de nouveaux concepts, mais à développer les raisonnements documentaires et les expériences lecteur qui permettront d'exploiter pleinement le patrimoine documentaire constitué.
