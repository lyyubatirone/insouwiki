# ARCHITECTURE DE L'EXPLORATION DOCUMENTAIRE

## Objet

Ce document décrit la manière dont un lecteur explore le patrimoine documentaire d'InsouWiki.

Contrairement aux documents d'architecture technique, il s'intéresse au parcours intellectuel proposé au lecteur.

L'objectif d'InsouWiki n'est pas de fournir une réponse unique.

Il est de permettre une exploration progressive, documentée et vérifiable d'un patrimoine documentaire.

---

# Principe fondamental

Une exploration documentaire commence toujours par une question.

Cette question conduit le lecteur vers un sujet documentaire.

À partir de ce sujet, InsouWiki propose progressivement différentes observations documentaires.

Le lecteur choisit ensuite le niveau de détail qu'il souhaite explorer.

Chaque niveau permet de revenir aux preuves ayant permis de produire les observations.

---

# Les niveaux d'exploration

L'exploration documentaire suit une progression naturelle.

```text
Question du lecteur
        │
        ▼
Sujet documentaire
        │
        ▼
Observations documentaires
        │
        ▼
Connaissances documentaires
        │
        ▼
Faits documentaires
        │
        ▼
Séquences documentaires
        │
        ▼
Transcription
        │
        ▼
Document d'origine
```

Chaque niveau apporte davantage de contexte.

Le lecteur peut interrompre son exploration à tout moment ou revenir vers les preuves plus détaillées.

---

# Les observations documentaires

Les observations constituent le premier niveau de lecture.

Elles permettent d'appréhender rapidement un sujet documentaire.

À ce stade, InsouWiki ne présente aucune interprétation.

Il expose uniquement ce qui est observable à partir des preuves disponibles.

Les premières observations documentaires sont :

* continuité ;
* évolution ;
* convergence ;
* divergence.

D'autres observations pourront enrichir progressivement cette bibliothèque.

---

# Le rôle des preuves

Chaque observation documentaire doit pouvoir être justifiée.

Le lecteur peut toujours accéder :

* aux connaissances documentaires ;
* aux faits documentaires ;
* aux séquences concernées ;
* à la transcription ;
* au document d'origine.

L'exploration ne masque jamais les preuves.

Elle les organise.

---

# Une exploration progressive

Tous les lecteurs n'ont pas les mêmes besoins.

Certains souhaitent une vue d'ensemble.

D'autres veulent examiner chaque preuve.

L'architecture d'InsouWiki permet de passer progressivement :

* d'une vision synthétique ;
* à une analyse détaillée ;
* jusqu'au document d'origine.

Cette progression respecte le rythme d'exploration choisi par le lecteur.

---

# Le rôle de l'intelligence artificielle

L'intelligence artificielle intervient uniquement pour faciliter cette exploration.

Elle contribue notamment à :

* transcrire les documents ;
* identifier des faits documentaires ;
* assister certains rapprochements documentaires.

Elle ne remplace jamais les preuves.

Elle ne remplace jamais le jugement du lecteur.

---

# Une architecture réversible

L'exploration documentaire est entièrement réversible.

À partir de n'importe quelle observation, le lecteur peut retrouver les preuves qui l'ont rendue possible.

Inversement, à partir d'un document, il peut découvrir les différentes observations documentaires auxquelles celui-ci contribue.

Cette réversibilité constitue une propriété essentielle d'InsouWiki.

---

# Vision

InsouWiki ne cherche pas à répondre à la place du lecteur.

Il construit un environnement documentaire dans lequel chaque observation peut être comprise, vérifiée et approfondie.

L'exploration documentaire devient ainsi un parcours intellectuel fondé sur les preuves plutôt qu'une succession de réponses produites par une intelligence artificielle.

Le rôle d'InsouWiki est d'organiser ce parcours afin de rendre un patrimoine documentaire vaste et complexe plus accessible, plus intelligible et plus facilement vérifiable.
