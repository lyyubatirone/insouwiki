# Cycle de vie d'un document dans InsouWiki

## Préambule

InsouWiki a pour vocation de constituer un patrimoine documentaire durable à partir de sources primaires.

Le projet ne cherche pas à remplacer les documents originaux mais à les enrichir progressivement, en conservant à chaque étape un lien permanent avec la preuve documentaire.

Chaque traitement réalisé par InsouWiki ajoute une nouvelle couche d'information sans jamais altérer les données précédentes.

Cette philosophie garantit la traçabilité complète des connaissances publiées.

---

# Le modèle documentaire

Le patrimoine documentaire d'InsouWiki est organisé autour de cinq objets fondamentaux.

```
Source
    │
    ▼
Point de diffusion
    │
    ▼
Document
    │
    ▼
Segment
    │
    ▼
Affirmation
```

Chaque niveau enrichit le précédent.

Chaque niveau conserve un lien permanent avec son origine.

Aucune information n'existe indépendamment de sa preuve documentaire.

---

# Les objets documentaires

## 1. Source

La source représente l'auteur documentaire.

Il peut s'agir :

* d'une personne ;
* d'une organisation ;
* d'une institution.

Exemples :

* Jean-Luc Mélenchon
* Institut La Boétie
* Assemblée nationale

Une source possède une identité permanente.

---

## 2. Point de diffusion

Le point de diffusion représente l'endroit où une source publie ses documents.

Exemples :

* une chaîne YouTube ;
* un site officiel ;
* un flux RSS ;
* un espace de podcasts.

Une même source peut posséder plusieurs points de diffusion.

---

## 3. Document

Le document est une œuvre documentaire unique.

Exemples :

* une vidéo ;
* un article ;
* un PDF ;
* un enregistrement audio.

Chaque document possède un identifiant documentaire permanent.

Le document constitue la référence documentaire de toutes les informations produites ultérieurement.

---

## 4. Segment

Le segment représente une partie cohérente d'un document.

Il peut être défini :

* par un intervalle temporel ;
* par une section d'un texte ;
* par une séquence logique.

Le segment facilite la navigation et constitue l'unité documentaire de travail.

---

## 5. Affirmation

L'affirmation représente une unité de connaissance extraite d'un segment.

Une affirmation n'est jamais isolée.

Elle est toujours reliée :

* à un segment ;
* à un document ;
* à un point de diffusion ;
* à une source.

Ainsi, toute information publiée peut être vérifiée à partir de son document primaire.

---

# Le cycle de vie documentaire

Le cycle de vie d'un document se déroule selon les étapes suivantes.

## Découverte

Le document est identifié depuis un point de diffusion.

Ses métadonnées minimales sont enregistrées.

---

## Enregistrement

Le document reçoit un identifiant documentaire permanent.

Il rejoint le patrimoine documentaire d'InsouWiki.

---

## Collecte des métadonnées

Les informations descriptives sont complétées :

* auteur ;
* date ;
* miniature ;
* plateforme ;
* identifiants externes.

---

## Transcription

Le contenu du document est converti en texte.

Cette transcription devient une nouvelle ressource documentaire.

Le document d'origine demeure inchangé.

---

## Segmentation

La transcription est découpée en passages cohérents.

Chaque segment conserve sa position dans le document d'origine.

---

## Résumé documentaire

Chaque segment reçoit un résumé neutre destiné à faciliter la lecture.

Le résumé constitue une aide à la navigation.

Il ne remplace jamais la transcription.

---

## Indexation

Les thèmes, mots-clés et entités sont identifiés.

Cette étape améliore les capacités de recherche.

---

## Extraction des affirmations

Les affirmations documentaires sont extraites des segments.

Chaque affirmation reste reliée à son contexte documentaire complet.

---

## Publication

Le document devient consultable.

Le lecteur peut naviguer :

* depuis la source ;
* vers le point de diffusion ;
* vers le document ;
* vers le segment ;
* jusqu'à l'affirmation.

À tout moment, il peut revenir au document primaire.

---

# Les principes fondateurs

InsouWiki respecte les principes suivants.

## 1. La preuve précède toujours l'interprétation

Toute connaissance publiée est reliée à une preuve documentaire.

---

## 2. Le patrimoine documentaire est immuable

Les traitements enrichissent le patrimoine.

Ils ne remplacent jamais les données existantes.

---

## 3. La traçabilité est permanente

Chaque information peut être remontée jusqu'au document primaire et, lorsque cela est possible, jusqu'à l'instant précis où elle apparaît.

---

## 4. Les enrichissements sont progressifs

Un document peut être découvert avant d'être transcrit.

Il peut être transcrit avant d'être segmenté.

Il peut être segmenté avant d'être résumé.

Chaque étape apporte une nouvelle valeur documentaire sans rendre les précédentes obsolètes.

---

## 5. La neutralité documentaire

InsouWiki décrit les documents.

Il ne modifie pas leur contenu.

Les traitements automatiques visent à faciliter l'accès à la connaissance, jamais à se substituer au jugement du lecteur.

---

# Conclusion

InsouWiki ne constitue pas seulement une collection de documents.

Il construit progressivement un patrimoine documentaire vérifiable.

Chaque enrichissement rapproche le lecteur de la connaissance tout en conservant, à chaque niveau, le lien avec la preuve documentaire originale.

Cette continuité entre la source primaire et la connaissance publiée constitue le principe fondateur d'InsouWiki.
