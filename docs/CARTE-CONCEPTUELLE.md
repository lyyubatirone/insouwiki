# CARTE CONCEPTUELLE D'INSOUWIKI

> *Comprendre l'ensemble du projet en quelques pages.*

---

# Objet

Cette carte conceptuelle présente l'organisation générale d'InsouWiki.

Elle ne décrit ni les choix techniques ni les détails d'implémentation.

Elle montre comment les principaux concepts du projet s'articulent entre eux.

Chaque document de la documentation développe ensuite l'une des parties de cette carte.

---

# 1. Pourquoi InsouWiki existe

```text
Le lecteur
        │
        ▼
Comprendre un sujet documentaire
        │
        ▼
Explorer des preuves
        │
        ▼
Construire son propre jugement
```

InsouWiki n'a pas été conçu pour produire des réponses définitives.

Il a été conçu pour rendre un patrimoine documentaire vaste, complexe et vérifiable plus facilement explorable.

---

# 2. Le patrimoine documentaire

```text
Sources
        │
        ▼
Documents
        │
        ▼
Transcriptions
        │
        ▼
Séquences documentaires
```

Le patrimoine documentaire constitue la matière première d'InsouWiki.

Aucune connaissance n'existe sans documents.

---

# 3. La construction de la connaissance

```text
Séquences documentaires
        │
        ▼
Faits documentaires
        │
        ▼
Connaissances documentaires
        │
        ▼
Observations documentaires
```

Chaque niveau synthétise progressivement le précédent.

À tout moment, le lecteur peut revenir aux preuves.

---

# 4. Le cœur documentaire

```text
                 Sujet documentaire
                        │
        ┌───────────────┼───────────────┐
        │               │               │
Observations     Connaissances     Documents
        │               │               │
        └───────────────┼───────────────┘
                        │
                 Faits documentaires
                        │
                 Séquences documentaires
                        │
                 Documents d'origine
```

Le sujet documentaire constitue le point de convergence de l'ensemble du patrimoine.

Il relie les documents, les faits, les connaissances et les observations.

---

# 5. Les raisonnements documentaires

```text
                Un auteur

        Continuité

        Évolution


            Plusieurs auteurs

        Convergence

        Divergence
```

Les raisonnements documentaires ne produisent jamais d'interprétation.

Ils mettent uniquement en évidence des observations fondées sur les preuves disponibles.

---

# 6. L'exploration documentaire

```text
Question
        │
        ▼
Sujet documentaire
        │
        ▼
Observations documentaires
        │
        ▼
Connaissances documentaires
        │
        ▼
Faits documentaires
        │
        ▼
Documents
```

L'exploration progresse toujours vers davantage de contexte.

Elle demeure entièrement réversible.

---

# 7. Les portes d'entrée

```text
Auteur

Sujet

Question

Citation

Document

        │

        ▼

Même patrimoine documentaire
```

Chaque lecteur peut commencer son exploration selon son propre besoin.

Toutes les portes conduisent vers le même patrimoine documentaire.

---

# 8. Les expériences documentaires

Le lecteur peut notamment :

* explorer un sujet ;
* explorer un auteur ;
* comparer plusieurs auteurs ;
* vérifier une observation documentaire.

Ces expériences sont indépendantes des interfaces qui les proposeront.

---

# 9. Les principes fondateurs

Toute l'architecture d'InsouWiki repose sur quelques principes simples.

* Les preuves précèdent toujours les raisonnements.
* Les raisonnements précèdent toujours les conclusions.
* L'intelligence artificielle facilite l'exploration.
* Le jugement appartient toujours au lecteur.
* Toute observation demeure traçable jusqu'aux documents d'origine.

---

# 10. La philosophie d'InsouWiki

```text
Documents

↓

Preuves

↓

Compréhension

↓

Liberté du lecteur
```

InsouWiki ne cherche pas à penser à la place du lecteur.

Il organise un patrimoine documentaire afin que chacun puisse construire sa compréhension à partir de preuves vérifiables.

La technologie n'a de valeur que lorsqu'elle rend le lecteur plus libre.
