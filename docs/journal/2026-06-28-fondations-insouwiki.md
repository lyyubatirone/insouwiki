# 2026-06-28 — Les fondations d'InsouWiki

Aujourd'hui, nous avons eu le sentiment d'achever la première grande étape d'InsouWiki.

Au début du projet, notre objectif paraissait simple : construire une base documentaire capable d'exploiter automatiquement des sources primaires.

Très vite, nous avons compris qu'il ne suffisait pas de développer un logiciel. Il fallait d'abord définir une méthode documentaire.

Pendant plusieurs semaines, nous avons donc volontairement ralenti.

Nous avons défini les notions fondamentales : document, source primaire, transcription, fait documentaire, traçabilité, patrimoine documentaire.

Nous avons écrit une charte, une vision, un cycle de vie documentaire et plusieurs décisions d'architecture avant même de connecter les premières technologies.

Cette démarche a parfois donné l'impression de retarder le développement.

Avec le recul, nous pensons au contraire qu'elle l'a rendu possible.

Aujourd'hui, l'architecture d'InsouWiki est suffisamment stable pour accueillir de nouvelles capacités documentaires sans remettre en cause ses fondements.

Nous avons également adopté plusieurs principes qui guideront durablement le projet :

* le domaine documentaire guide toujours les choix techniques ;
* chaque composant possède une responsabilité unique ;
* toute dépendance externe est isolée derrière une interface ;
* chaque nouvelle capacité est testée avant d'être intégrée ;
* l'architecture n'évolue que lorsqu'un besoin réel apparaît.

Nous avons également découvert une manière de travailler qui nous ressemble.

Avant de coder, nous cherchons à comprendre.

Avant d'ajouter une fonctionnalité, nous vérifions qu'elle trouve naturellement sa place dans l'architecture.

Avant de réorganiser le projet, nous nous demandons si cette évolution apporte une valeur réelle.

Cette discipline demande parfois un peu plus de temps.

Nous pensons qu'elle permettra au projet de rester compréhensible, maintenable et fidèle à sa vocation pendant de nombreuses années.

Aujourd'hui, nous considérons que les fondations d'InsouWiki sont posées.

Le projet dispose désormais :

* d'une vision ;
* d'une charte documentaire ;
* d'un modèle documentaire ;
* d'une architecture applicative ;
* d'interfaces clairement définies ;
* de services testés ;
* d'une méthode de développement cohérente.

Les prochaines étapes seront différentes.

Nous allons progressivement relier ces fondations au monde réel : extraction audio, transcription, segmentation, puis production de faits documentaires.

Nous avons le sentiment de quitter la phase de conception pour entrer dans celle de la construction.

La première transcription réelle ne marquera pas seulement une réussite technique.

Elle sera le premier document à parcourir entièrement le cycle documentaire imaginé depuis le début de cette aventure.

---

*"Les fondations d'un projet ne sont pas ce qui se voit le plus. Elles sont ce qui permet à tout le reste de durer."*
